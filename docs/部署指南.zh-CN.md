# Claude Code 部署指南

本文档说明如何将 Claude Code 项目打包成可复制的产品，并部署到其他终端使用。

## 概述

Claude Code 是一个基于 Bun 运行时的 CLI 工具，采用代码分割（code splitting）构建。默认构建配置生成 `dist/` 目录，包含主入口 `cli.js` 和约 450 个 chunk 文件。

## 系统要求

| 平台 | 最低要求 |
|------|----------|
| **Windows** | Bun 1.0+（建议使用最新版本） |
| **macOS** | Bun 1.0+ |
| **Linux** | Bun 1.0+ |

### 推荐配置

| 设备类型 | Bun 版本 | Node.js 兼容性 |
|----------|------------|
| 本地开发 | Bun 最新 | 完全兼容 |
| 单机部署 | Bun 1.0+ | 完全兼容 |

## 安装 Bun

### Windows

```powershell
# 使用 PowerShell 安装脚本
irm bun.sh/install.ps1 | iex
```

### macOS / Linux

```bash
curl -fsSL https://bun.sh/install | bash
```

### 验证安装

```bash
bun --version
```

## 构建项目

### 标准构建

```bash
# 克隆项目
git clone <repository-url> claude-code
cd claude-code

# 安装依赖
bun install

# 构建项目
bun run build
```

### 使用默认配置构建

项目默认包含配置文件模板。如果需要自定义配置，可以在构建前修改或使用环境变量。

## 部署方式

### 方式一：直接运行（开发/测试环境）

适用于开发环境或需要频繁更新的场景。

```bash
# 克隆项目
git clone <repository-url> claude-code
cd claude-code

# 安装依赖
bun install

# 运行
bun run dev [参数]
```

### 方式二：复制构建产物（推荐）

适用于简单部署场景，无需自定义配置。

```bash
# 1. 构建项目
bun run build

# 2. 创建部署目录
mkdir -p claude-deploy
cd claude-deploy

# 3. 复制构建产物
# 包含 dist 目录和必要的依赖
cp -r dist claude-deploy/
cp package.json claude-deploy/

# 4. 可选：复制配置文件
# cp ~/.claude/settings.json claude-deploy/.claude/

# 5. 可选：复制本地模型相关文件
# 如果使用 OpenAI 兼容模式，需要确保模型文件可用

# 6. 在目标机器上运行
cd claude-deploy
bun cli.js
```

### 方式三：源码部署（推荐）

适用于需要自定义配置或频繁更新的场景。

```bash
# 1. 复制源码到目标机器
# 使用 scp、rsync 或 git 等方式

# 2. 在目标目录安装依赖并构建
cd claude-code
bun install
bun run build

# 3. 创建配置文件
mkdir -p ~/.claude
cp examples/配置示例.zh-CN.json ~/.claude/settings.json

# 4. 运行
bun cli.js
```

### 方式四：修改构建配置，生成单文件包（推荐）

修改 `build.ts`，让依赖内联：

```typescript
// 在 build.ts 中修改 Bun.build 配置
const result = await Bun.build({
  entrypoints: ["src/entrypoints/cli.tsx"],
  outdir: "dist-standalone",
  target: "node", // 改为 node
  splitting: false, // 关闭代码分割
  packages: "bundle", // 内联依赖
  define: getMacroDefines(),
  features,
});
```

**步骤**：
1. 备份原 `build.ts`
2. 应用上述修改
3. 重新构建：`bun run build`
4. 测试：`node dist-standalone/cli.js --version`

**优势**：
- 生成单个文件，便于分发
- 包含所有依赖，无需外部 `node_modules`
- 可在任何有 Node.js 的环境运行
- 减小部署包体积

### 方式五：使用 pkg 打包成可执行文件

```bash
# 安装 pkg
npm install -g pkg

# 创建 package.json 指定入口
echo '{"bin": "cli.js"}' > package.json

# 打包（需指定 node 版本）
pkg cli.js --target node18-win-x64 --output claude-code.exe
```

## 配置管理

### 创建配置文件

在 `~/.claude/settings.json` 中创建配置文件：

```json
{
  "$schema": "https://json.schemastore.org/claude-code-settings.json",
  "apiProviderConfig": {
    "provider": "openai",
    "blockAnthropicAPI": true,
    "forceProvider": true,
    "customEndpoints": {
      "openAIBaseURL": "https://api.openai.com/v1"
    }
  },
  "privacyLevel": "essential-traffic",
  "documentationLanguage": "zh-CN",
  "forceDocumentationLanguage": true,
  "permissions": {
    "defaultMode": "ask"
  }
}
```

### 配置文件位置优先级

Claude Code 按照以下顺序查找配置文件：

1. **策略配置**：`managed-settings.json`（企业管理员配置，最高优先级）
2. **本地配置**：`.claude/settings.local.json`（项目本地设置）
3. **项目配置**：`.claude/settings.json`（项目特定设置）
4. **用户配置**：`~/.claude/settings.json`（用户全局设置，默认位置）

## 验证部署

### 检查配置加载

```bash
# Linux/macOS
claude --dump-config

# Windows PowerShell
.\claude.exe --dump-config
```

### 验证 API 屏蔽

如果配置了 `blockAnthropicAPI`，尝试访问 `api.anthropic.com` 时应该被重定向或阻止。

### 验证隐私级别

检查网络流量，确认没有发送遥测数据到 Anthropic。

```bash
# Linux/macOS
sudo tcpdump -i any -n host api.anthropic.com

# Windows 资源监视器
# 使用系统自带的资源监视器或 Wireshark
```

### 验证文档语言

查看文档加载是否使用配置的语言版本。

## 常见问题

### 构建产物无法在其他机器运行

**原因**：构建产物包含绝对路径或平台特定的二进制文件。

**解决方法**：
1. 使用源码部署方式（推荐）
2. 或使用 Docker 容器化部署
3. 确保目标机器安装相同版本的 Bun

### API 请求失败

**可能原因**：
1. API 配置不正确
2. 网络连接问题
3. API 密钥无效

**解决方法**：
1. 检查 `~/.claude/settings.json` 配置
2. 验证 API 端点和密钥
3. 查看应用程序日志中的错误信息

### 配置文件不生效

**原因**：配置文件格式错误或位置不正确。

**解决方法**：
1. 验证 JSON 格式是否正确
2. 确认所有字符串和布尔值格式正确
3. 查看是否有语法错误（多余的逗号、引号等）

### 权限问题

**原因**：配置了不正确的权限模式。

**解决方法**：
1. 检查 `permissions.defaultMode` 配置
2. 确认权限对话框中的行为符合预期
3. 查看环境变量 `CLAUDE_CODE_SIMPLE` 是否影响权限模式

## 技术说明

### 配置优先级
- 策略配置 > 本地配置 > 项目配置 > 用户配置
- 配置文件中的设置会覆盖环境变量
- `privacyLevel` 在配置文件中的优先级高于 `DISABLE_TELEMETRY` 和 `CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC` 环境变量

### API 屏蔽机制
- 当 `blockAnthropicAPI` 为 `true` 时，所有到 `api.anthropic.com` 的请求将被拦截
- 如果配置了 `customEndpoints.openAIBaseURL`，请求将被静默重定向到指定端点
- 重定向会自动映射端点路径（例如 `/v1/messages` → `/v1/chat/completions`）
- 认证头会自动转换为 OpenAI 格式

### 隐私级别影响
- `essential-traffic`：禁用遥测、自动更新、功能检查、发布说明等
- `no-telemetry`：仅禁用遥测数据收集
- 修改隐私级别后需要重启应用程序才能生效

### 文档系统
- 支持多语言文档自动发现
- 默认回退到英文文档
- 可配置强制使用指定语言
- 支持的语言代码：`en`、`zh-CN`、`zh-TW`

## 最佳实践

1. **始终使用 JSON Schema 验证**：配置文件应包含 `$schema` 引用
2. **分环境配置**：为不同的使用场景创建不同的配置文件
3. **版本控制配置**：使用 Git 管理配置文件的变更
4. **文档化配置**：为中文用户创建 `zh-CN` 版本配置示例
5. **测试配置**：在测试环境中验证配置行为后再部署
