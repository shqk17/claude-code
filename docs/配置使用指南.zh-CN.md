# Claude Code 配置使用指南

## 快速开始

1. 创建配置文件：`~/.claude/settings.json`
2. 复制示例配置
3. 重启Claude Code

## 配置选项详解

### API提供商配置

通过 `apiProviderConfig` 节点配置API提供商和屏蔽Anthropic API：

| 选项                                 | 类型      | 描述                                                                     |
| ---------------------------------- | ------- | ---------------------------------------------------------------------- |
| `provider`                         | string  | 强制使用的API提供商：`anthropic`、`openai`、`bedrock`、`vertex`、`foundry`、`custom` |
| `blockAnthropicAPI`                | boolean | 是否屏蔽所有到 `api.anthropic.com` 的请求                                        |
| `forceProvider`                    | boolean | 是否强制使用指定提供商，忽略环境变量                                                     |
| `customEndpoints.openAIBaseURL`    | string  | OpenAI兼容端点的基础URL                                                       |
| `customEndpoints.anthropicBaseURL` | string  | Anthropic API的基础URL                                                    |
| `customEndpoints.bedrockRegion`    | string  | AWS Bedrock区域                                                          |

**示例：完全使用OpenAI**

```json
{
  "apiProviderConfig": {
    "provider": "openai",
    "blockAnthropicAPI": true,
    "forceProvider": true,
    "customEndpoints": {
      "openAIBaseURL": "https://api.openai.com/v1"
    }
  }
}
```

**示例：使用本地模型（Ollama）**

```json
{
  "apiProviderConfig": {
    "provider": "openai",
    "blockAnthropicAPI": true,
    "customEndpoints": {
      "openAIBaseURL": "http://localhost:11434/v1"
    }
  }
}
```

### 隐私级别

通过 `privacyLevel`` 控制数据收集和网络流量：

| 级别                  | 描述                         |
| ------------------- | -------------------------- |
| `default`           | 默认模式，启用所有功能                |
| `no-telemetry`      | 禁用遥测（Datadog、1P事件、反馈调查）    |
| `essential-traffic` | 禁用所有非必要网络流量（遥测+自动更新、功能检查等） |

**示例：**

```json
{
  "privacyLevel": "essential-traffic"
}
```

**环境变量支持：**
配置文件优先级高于环境变量。如果配置文件中设置了隐私级别，环境变量将被忽略。这提供了向后兼容性。

### 文档语言

通过 `documentationLanguage`` 和 `forceDocumentationLanguage`` 配置文档语言偏好：

| 选项                           | 类型      | 描述                                        |
| ---------------------------- | ------- | ----------------------------------------- |
| `documentationLanguage`      | string  | 文档语言：`en`（英文）、`zh-CN`（简体中文）、`zh-TW`（繁体中文） |
| `forceDocumentationLanguage` | boolean | 是否强制使用指定语言，不进行英文回退                        |

**示例：**

```json
{
  "documentationLanguage": "zh-CN",
  "forceDocumentationLanguage": true
}
```

### 权限配置

通过 `permissions.defaultMode`` 配置默认权限模式：

| 模式       | 描述                |
| -------- | ----------------- |
| `ask`    | 遇到每个操作都询问用户确认     |
| `auto`   | 自动批准所有操作（需要信任对话框） |
| `always` | 允许所有操作，无需确认       |

**示例：**

```json
{
  "permissions": {
    "defaultMode": "ask"
  }
}
```

## 完整配置示例

### 完全隔离配置（推荐）

适用于需要完全脱离Anthropic服务的场景：

```json
{
  "$schema": "https://json.schemastore.org/claude-code-settings.json",
  "apiProviderConfig": {
    "provider": "openai",
    "blockAnthropicAPI": true,
    "forceProvider": true,
    "customEndpoints": {
      "openAIBaseURL": "http://localhost:11434/v1"
    }
  },
  "privacyLevel": "essential-traffic",
  "documentationLanguage": "zh-CN",
  "forceDocumentationLanguage": true,
  "permissions": {
    "defaultMode": "ask"
  }
}
```

### 仅禁用遥测配置

适用于希望继续使用Anthropic API但禁用数据收集的场景：

```json
{
  "privacyLevel": "no-telemetry"
}
```

### 使用第三方OpenAI兼容API

适用于使用DeepSeek、阿里云等第三方服务的场景：

```json
{
  "apiProviderConfig": {
    "provider": "openai",
    "customEndpoints": {
      "openAIBaseURL": "https://api.deepseek.com/v1"
    }
  }
}
```

## 配置文件位置

Claude Code 按照以下优先级查找配置文件：

1. **策略配置**：`managed-settings.json`（企业管理员配置，最高优先级）
2. **本地配置**：`.claude/settings.local.json`（项目本地设置）
3. **项目配置**：`.claude/settings.json`（项目特定设置）
4. **用户配置**：`~/.claude/settings.json`（用户全局设置，默认位置）

## 验证配置

### 检查配置加载

```bash
# Linux/macOS
claude --dump-config

# Windows PowerShell
.\claude.exe --dump-config
```

### 验证API屏蔽

启动应用程序并尝试访问 `api.anthropic.com`，如果配置正确，请求将被重定向或阻止。

### 验证隐私级别

检查网络流量：

```bash
# Linux/macOS
sudo tcpdump -i any -n host api.anthropic.com

# Windows 资源监视器
# 使用系统自带的资源监视器或Wireshark
```

### 验证文档语言

查看文档加载是否使用配置的语言版本。

## 常见问题

### API屏蔽不生效

1. 确认配置文件路径正确
2. 检查 `blockAnthropicAPI` 设置为 `true`
3. 重启应用程序
4. 检查日志中是否有初始化错误

### 配置验证失败

1. 检查 JSON 格式是否正确
2. 确认所有字符串和布尔值格式正确
3. 查看是否有语法错误（多余的逗号、引号等）

### 权限模式问题

1. 检查 `permissions.defaultMode` 配置
2. 确认权限对话框中的行为符合预期
3. 查看环境变量 `CLAUDE_CODE_SIMPLE` 是否影响权限模式

## 技术说明

### 配置优先级

- 策略配置 > 本地配置 > 项目配置 > 用户配置
- 配置文件中的设置会覆盖环境变量
- `privacyLevel` 在配置文件中的优先级高于 `DISABLE_TELEMETRY` 和 `CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC` 环境变量

### API屏蔽机制

- 当 `blockAnthropicAPI` 为 `true` 时，所有到 `api.anthropic.com` 的请求将被拦截
- 如果配置了 `customEndpoints.openAIBaseURL`，请求将被静默重定向到指定端点
- 重定向会自动映射端点路径（例如 `/v1/messages` → `/v1/chat/completions`）
- 认证头会自动转换为OpenAI格式

### 隐私级别影响

- `essential-traffic`：禁用遥测、自动更新、功能检查、发布说明等
- `no-telemetry`：仅禁用遥测数据收集
- 修改隐私级别后需要重启应用程序才能生效

### 文档系统

- 支持多语言文档自动发现
- 默认回退到英文文档
- 可配置强制使用指定语言
- 支持的语言代码：`en`、`zh-CN`、`zh-TW`

## 最佳实践

1. **始终使用JSON Schema验证**：配置文件应包含 `$schema` 引用
2. **分环境配置**：为不同的使用场景创建不同的配置文件
3. **版本控制配置**：使用Git管理配置文件的变更
4. **文档化配置**：为中文用户创建 `zh-CN` 版本配置示例
5. **测试配置**：在测试环境中验证配置行为后再部署
